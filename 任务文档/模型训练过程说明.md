# 模型训练过程说明

## 一、文档目的

本文档用于说明本项目中“酒店预订取消预测模型”的训练过程、特征工程、模型对比、最终模型选择和与老师机器学习要求的对应关系。后续提交代码时，本文件可作为成员 A 机器学习模块的说明材料，也可在课程报告和答辩中复用。

## 二、机器学习任务定义

本项目的机器学习任务是预测酒店预订是否会被取消，属于有标签的监督学习分类任务。

| 项目 | 内容 |
|---|---|
| 数据文件 | `数据/cleaned_hotel_bookings.csv` |
| 样本 | 每一条酒店预订记录 |
| 特征 X | 24 个酒店预订相关字段 |
| 标签 y | `is_canceled` |
| 标签含义 | 0 表示未取消，1 表示已取消 |
| 任务类型 | 二分类 Classification |
| 训练脚本 | `scripts/train_model.py` |
| 最终模型文件 | `models/cancellation_model.pkl` |
| 特征列文件 | `models/feature_columns.json` |
| 指标文件 | `models/metrics.json` |

根据老师《机器学习基础》中的分类标准，本项目的标签是“是 / 否”类型，即“是否取消”，因此不适合做回归或聚类，而应选择分类模型。

## 五、特征工程处理

本次模型训练做了基础特征工程，但没有改动字段规范和接口契约。

### 5.1 数值特征处理

数值型字段包括提前预订天数、入住晚数、住客人数、历史取消次数、房价、停车位需求数、特殊需求数量等。

处理方式：

1. 使用中位数填补缺失值。
2. 使用标准化处理数值范围。
3. 将处理后的数值特征输入模型。

对应字段包括：

```text
lead_time
total_nights
adults
children
babies
total_guests
is_repeated_guest
previous_cancellations
previous_bookings_not_canceled
room_type_changed
booking_changes
days_in_waiting_list
adr
required_car_parking_spaces
total_of_special_requests
```

### 5.2 分类特征处理

分类字段包括酒店类型、餐食类型、国家地区、市场渠道、分销渠道、房型、订金类型、客户类型等。

处理方式：

1. 使用众数填补缺失值。
2. 使用 One-Hot 编码把文本类别转换为模型可识别的数值特征。
3. 设置 `handle_unknown="ignore"`，保证后续预测时遇到训练集中未出现的新类别不会直接报错。

对应字段包括：

```text
hotel
meal
country_code
market_segment
distribution_channel
reserved_room_type
assigned_room_type
deposit_type
customer_type
```

### 5.3 训练流水线

训练脚本使用 Scikit-learn 的 `Pipeline` 和 `ColumnTransformer`，把“缺失值处理、标准化、One-Hot 编码、分类模型”封装成完整流水线。

这样保存的 `cancellation_model.pkl` 不是单独的分类器，而是包含预处理步骤和分类器的完整模型流水线。后续后端预测时，只要传入这 24 个原始特征字段，就可以直接调用模型进行预测。

## 六、训练集与测试集划分

训练脚本使用 8:2 的比例划分训练集和测试集。

| 数据集 | 作用 |
|---|---|
| 训练集 | 模型学习规律，相当于老师课件中“课本和练习册” |
| 测试集 | 检验模型在没见过的数据上的效果，相当于“期末考试卷” |

这种做法符合老师要求，避免只在全部数据上训练再直接评价造成“自欺欺人”的情况。训练和测试划分时设置了 `random_state=42`，保证每次运行结果可复现；在标签类别满足条件时使用分层划分，尽量保持训练集和测试集中取消 / 未取消比例一致。

## 七、模型选择与对比

根据老师要求，模型选择需要根据标签类型决定。本项目标签是是否取消，因此选择分类模型进行对比。

本次训练对比了两个模型：

| 模型 | 选择原因 |
|---|---|
| 逻辑回归 Logistic Regression | 分类任务中的基础模型，训练快，便于作为基线模型解释 |
| 随机森林 Random Forest | 能处理多种特征和非线性关系，稳定性较好，适合酒店预订取消这种受多因素影响的问题 |

训练脚本会分别训练逻辑回归和随机森林，并在测试集上计算评估指标。最终选择规则是：优先比较 F1 值，F1 值更高者优先；如果 F1 值接近，再比较准确率。

## 八、模型评估指标

本项目是二分类任务，因此没有只看准确率，而是同时记录以下指标：

| 指标 | 含义 | 本项目用途 |
|---|---|---|
| Accuracy 准确率 | 全部样本中预测正确的比例 | 衡量整体预测正确率 |
| Precision 精确率 | 预测为取消的订单中，实际取消的比例 | 衡量高风险判断是否可靠 |
| Recall 召回率 | 实际取消订单中，被模型识别出来的比例 | 衡量模型是否漏掉取消订单 |
| F1 值 | 精确率和召回率的综合指标 | 用于选择最终模型 |
| Confusion Matrix 混淆矩阵 | 展示预测对 / 预测错的数量分布 | 用于页面和报告解释分类结果 |
| Train Score 训练集得分 | 模型在训练集上的表现 | 用于判断是否欠拟合或过拟合 |
| Test Score 测试集得分 | 模型在测试集上的表现 | 用于判断泛化能力 |

这符合老师新增要求中“不能只写一个准确率，应展示模型对比、测试结果、混淆矩阵和训练集 / 测试集表现”的要求。

## 九、模型对比结果

当前真实训练结果如下，数据来自 `models/metrics.json`。

| 模型 | Accuracy | Precision | Recall | F1 | Train Score | Test Score |
|---|---:|---:|---:|---:|---:|---:|
| 逻辑回归 | 0.8115 | 0.7256 | 0.7898 | 0.7563 | 0.8126 | 0.8115 |
| 随机森林 | 0.8707 | 0.8249 | 0.8263 | 0.8256 | 0.9113 | 0.8707 |

逻辑回归的整体表现可以作为基线模型。随机森林在准确率、精确率、召回率和 F1 值上都优于逻辑回归，因此最终选择随机森林作为取消预测模型。

## 十、最终采用模型

最终采用模型为：

```text
random_forest_v1
```

选择理由：

1. 本项目是是否取消的二分类任务，随机森林适用于分类问题。
2. 酒店预订取消受提前预订天数、订金类型、客户类型、历史取消次数、特殊需求数量等多因素影响，关系可能不是简单线性关系。
3. 随机森林能组合多棵决策树，对复杂特征关系有更强表达能力。
4. 与逻辑回归相比，随机森林在测试集上的 F1 值更高。
5. 随机森林测试集准确率达到 0.8707，F1 值达到 0.8256，综合表现更好。

## 十一、过拟合与欠拟合判断

随机森林当前训练集得分为 0.9113，测试集得分为 0.8707。

| 项目 | 分数 |
|---|---:|
| 训练集得分 | 0.9113 |
| 测试集得分 | 0.8707 |
| 差值 | 0.0406 |

解释：

- 如果训练集和测试集都很低，说明模型可能欠拟合。
- 如果训练集远高于测试集，说明模型可能严重过拟合。
- 当前训练集得分略高于测试集得分，属于正常情况，说明模型学习到了一定规律，同时在测试集上仍保持较好表现。

因此，当前模型没有表现出明显严重过拟合，也不是欠拟合状态。

## 十二、超参数说明

训练脚本中对模型设置了基础超参数，保证模型稳定、可复现且训练时间可控。

### 12.1 逻辑回归

| 参数 | 当前值 | 说明 |
|---|---|---|
| `max_iter` | 1000 | 增加最大迭代次数，保证模型更容易收敛 |
| `class_weight` | `balanced` | 处理取消 / 未取消类别比例不完全均衡的问题 |
| `random_state` | 42 | 保证结果可复现 |

### 12.2 随机森林

| 参数 | 当前值 | 说明 |
|---|---|---|
| `n_estimators` | 120 | 使用 120 棵树，兼顾效果和训练速度 |
| `min_samples_leaf` | 2 | 限制叶子节点最小样本数，降低过拟合风险 |
| `class_weight` | `balanced` | 处理类别不均衡问题 |
| `random_state` | 42 | 保证结果可复现 |
| `n_jobs` | -1 | 使用可用 CPU 并行训练，提高训练速度 |

本次没有进行复杂网格搜索调参，原因是课程项目当前阶段优先保证训练流程完整、指标可解释、产物可接入。后续如果需要进一步提高模型效果，可以在此基础上增加 `max_depth`、`n_estimators`、`min_samples_leaf` 等参数的对比实验。
