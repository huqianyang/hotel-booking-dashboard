# 接口契约说明书

## 一、文档目的

本文档用于统一“酒店预订数据可视化分析与预测系统”的前后端接口口径，作为成员 A、B、C 并行开发和后续联调的接口契约。

- 成员 A 根据预测接口、模型评估接口确定模型输入输出格式。
- 成员 B 根据本文档实现 Flask API、数据库查询、模型调用和实时数据接口。
- 成员 C 根据本文档使用 mock 数据开发页面，后续切换到真实 API。

本文档先覆盖阶段 5 到阶段 9 的核心接口：首页、数据查询页、数据可视化页、评估与预测页，以及 Redis/Flume/Kafka/Storm 接入前的最小实时接口占位；后续进入完整实时链路阶段后再细化 Storm 写入和 Redis Key 结构。

## 二、通用约定

### 2.1 基础路径

```text
/api
```

所有业务接口统一以 `/api` 开头。

### 2.2 返回格式

成功返回：

```json
{
  "success": true,
  "data": {},
  "message": "ok"
}
```

失败返回：

```json
{
  "success": false,
  "data": null,
  "message": "错误说明"
}
```

### 2.3 分页格式

列表接口统一使用以下分页参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| page | int | 否 | 1 | 当前页码，从 1 开始 |
| page_size | int | 否 | 20 | 每页记录数 |

分页返回格式：

```json
{
  "items": [],
  "pagination": {
    "page": 1,
    "page_size": 20,
    "total": 119390,
    "total_pages": 5970
  }
}
```

### 2.4 日期格式

| 类型 | 格式 | 示例 |
|---|---|---|
| 日期 | YYYY-MM-DD | 2017-01-14 |
| 日期时间 | YYYY-MM-DD HH:mm:ss | 2017-01-14 12:30:00 |

### 2.5 风险等级

| risk_level | 中文含义 | 建议概率区间 |
|---|---|---|
| low | 低风险 | cancel_probability < 0.30 |
| medium | 中风险 | 0.30 <= cancel_probability < 0.60 |
| high | 高风险 | cancel_probability >= 0.60 |

页面展示时转换为：低风险、中风险、高风险。

## 三、页面路由契约

页面路由由 Flask 模板负责，返回 HTML。

| 页面 | URL | 负责人 | 说明 |
|---|---|---|---|
| 首页 | `/` | B/C | 系统概览、核心指标、实时状态入口 |
| 数据查询页 | `/bookings` | B/C | 订单筛选、列表、详情、编辑、逻辑删除 |
| 数据可视化页 | `/visualization` | B/C | 历史图表、地图联动、订单摘要 |
| 评估与预测页 | `/prediction` | A/B/C | 订单预测、模型评估、批量预测记录 |
| 健康检查 | `/health` | B | 返回服务状态 JSON |

## 四、首页接口

### 4.1 首页概览接口

```text
GET /api/dashboard/summary
```

#### 请求参数

无。

#### 返回示例

```json
{
  "success": true,
  "data": {
    "total_bookings": 119390,
    "canceled_bookings": 44224,
    "cancel_rate": 0.3704,
    "avg_adr": 101.83,
    "high_risk_count": 1280,
    "latest_event_time": "2017-01-14 14:00:00"
  },
  "message": "ok"
}
```

#### 字段说明

| 字段 | 中文含义 | 来源 |
|---|---|---|
| total_bookings | 总订单数 | MySQL `hotel_bookings` |
| canceled_bookings | 已取消订单数 | MySQL `hotel_bookings` |
| cancel_rate | 取消率 | 后端聚合计算 |
| avg_adr | 平均每日房价 | MySQL `hotel_bookings.adr` |
| high_risk_count | 高风险订单数 | 预测结果或模拟指标 |
| latest_event_time | 最新模拟业务时间 | Redis 或 MySQL 最新指标 |

#### 负责人

- B：实现后端聚合。
- C：展示指标卡。
- A：确认高风险口径是否与模型输出一致。
